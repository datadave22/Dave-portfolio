PROMPT START

You are an elite product design + engineering studio building a high-end personal brand website for a senior professional.

This is not a portfolio template.
This is a brand object.

The visual quality, motion design, layout discipline, copy tone, and system architecture must reflect:

- Apple-level product polish
- High-fashion editorial restraint
- Architectural clarity
- Modern SaaS-level technical credibility

Every design decision must feel intentional, restrained, and premium.

────────────────────────────
1. Project Purpose
────────────────────────────

This website is a personal brand landing experience designed for:

- Potential employers
- High-value clients
- Creative directors
- Senior engineers and technical leads

The site must communicate:

- Taste
- Precision
- Thoughtfulness
- Technical credibility
- Seniority

It functions simultaneously as:

- A marketing surface
- A living professional journal / updates feed
- A central hub for resume, projects, and contact

This site should feel like something a creative director would bookmark and a hiring manager would remember.

────────────────────────────
2. Visual & Interaction Inspiration
────────────────────────────

Primary inspiration reference (behavioral, not literal):
https://www.cosmos.so/

Key distinction from the reference:

- NO image carousels
- NO sliding image galleries

Instead, use:

- Abstract geometric shapes
- Sacred-geometry–inspired structure (not decoration)
- Minimal architectural forms

Geometry must inform:

- Proportion
- Spacing
- Alignment
- Rhythm

NOT:

- Literal symbols
- Decorative spirals
- Mystical or ornamental visuals

Approved forms:

- Lines
- Grids
- Circles
- Rectangles
- Modular blocks
- Golden-ratio–inspired spacing (implicit, not visualized)

────────────────────────────
3. Motion Philosophy (CRITICAL)
────────────────────────────

Motion must feel:

- Smooth
- Slow
- Intentional
- Calm but confident

Apple-style easing only.
No gimmicks. No flashy effects.

Motion Rules:

- One primary motion idea per section
- Subtle opacity + position transitions (Y-axis preferred)
- No chaotic parallax
- No constant background movement
- Motion reinforces hierarchy, never distracts

Scroll Behavior:

- Scroll-based transitions at section boundaries
- Movement feels anchored and architectural
- Avoid continuous floating or jitter

Hover Behavior:

- Minimal, restrained feedback
- Slight opacity, scale, or line emphasis only
- No playful or exaggerated responses

Accessibility:

- Respect prefers-reduced-motion
- Motion degrades gracefully

────────────────────────────
4. Color System
────────────────────────────

Use a corporate-professional, neutral-first palette.

Base:
- Off-white / soft gray backgrounds

Primary:
- Deep charcoal / near-black

Accents:
- Muted metallics
- Warm gray
- Cool slate
- Subtle gold or blue-gray

Avoid:
- Neon colors
- Loud gradients
- Trend-driven palettes

Color transitions must be:
- Gentle fades
- No harsh contrast jumps
- WCAG contrast compliant at all times

────────────────────────────
5. Typography
────────────────────────────

- One primary modern sans-serif (editorial, professional)
- One optional secondary only if absolutely necessary

Typography principles:
- Strong hierarchy
- Generous whitespace
- Calm line lengths
- Editorial pacing

No decorative or playful fonts.

────────────────────────────
6. Layout & Content Structure
────────────────────────────

Required Sections:

Hero:
- Abstract geometric motion
- Personal name or brand mark
- Short, confident positioning statement
- No long paragraphs

About / Philosophy:
- Minimal copy
- Declarative, confident tone
- Designed like an editorial spread

Updates / Blog:
- Chronological
- Clean list or cards
- Reads as “signals of thinking,” not casual blogging

Projects / Work:
- Modular
- Expandable
- No clutter
- Clarity over quantity

Resume / Credentials:
- Clean timeline
- Downloadable
- Professional formatting

Contact:
- Clear
- Minimal friction
- Professional tone

────────────────────────────
7. Content Voice & Copy Rules
────────────────────────────

Copy must be:
- Declarative
- Calm
- Confident
- Precise

Avoid:
- Buzzwords
- Hype language
- Startup clichés
- Motivational fluff

Tone should feel like:
- An editorial caption
- A senior internal memo
- A design-forward personal manifesto

────────────────────────────
8. Technical Stack & Architecture
────────────────────────────

Framework:
- Modern frontend framework (React / Next.js preferred)

Architecture:
- Component-driven
- Design-system–first approach

Design system requirements:
- Consistent spacing scale
- Color tokens
- Typography tokens
- Motion tokens (shared easing + durations)

Animation:
- GPU-friendly
- Scroll-based
- No jank
- Performance-first

────────────────────────────
9. Git & Branching Strategy (MANDATORY)
────────────────────────────

Initialize Git immediately with the following branches:

- main → production only
- dev → stable development
- dev-ai → experimental / AI-assisted changes
- feature/* → isolated feature work

Rules:
- No direct commits to main
- Clear, descriptive commit messages
- Environment parity between branches
- Do not proceed until branches are created

────────────────────────────
10. Environment & Configuration
────────────────────────────

Set up environment configuration from the start:

- .env.example
- .env.local

Support environment separation:
- Development
- Production

Prepare for:
- Database integration
- Blog schema
- Projects schema

────────────────────────────
11. Database & Schema (Initial)
────────────────────────────

Design future-proof schemas even if data is mocked.

Blog / Updates:
- id
- title
- slug
- content
- published_at
- status (draft / published)
- tags

Projects:
- id
- name
- description
- role
- tech_stack
- featured

Use migrations or schema files from the start.

────────────────────────────
12. Quality Bar (NON-NEGOTIABLE)
────────────────────────────

This site must feel like:
- A senior designer/engineer’s personal site
- A high-end brand object
- Something built with restraint and taste

Rules:
- No cheap placeholders
- No default templates
- No rushed decisions

If something cannot be executed at a high standard:
- Do it minimally
- Do it cleanly
- Or do not do it at all

The site must still feel premium even with minimal content.
